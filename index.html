On the Relationship Between Code Review and Continuous Integration: An Exploratory Study

This website contains complementary data to the submitted paper.

Abstract

Code Review (CR) and Continuous Integration (CI) are two practices of Modern Software Engineering that have been widely disseminated and have generated good results in open-source and closed-source projects. Currently, some tools offer the possibility to work with these practices in an integrated way through the integration request mechanisms. This work aims to conduct an exploratory study to investigate the relationship between aspects of CR and CI in closed-source projects. This study aims to find out if there is a relationship between CR and CI, about: (i) CI workload, (ii) number of failures reported by CI, (iii) time to resolve errors raised by CI, and (iv) intensity of commits in the CR. Also, we investigate how CI can assist the CR process in software projects. We quantitatively analyzed 10 closed-source projects by collecting and analyzing 28 metrics related to CR and CI. We carry out a qualitative analysis by capturing the code reviewers' perceptions in the projects. From the metrics collected, we identified four significant correlations between the metrics of CR and CI. Our findings indicate that: (i) number of CRs can profoundly influence the workload generated for CI, (ii) number of CRs has a moderate impact on the number of failures pointed out by CI, (iii) total CR time strongly influences the time needed to correct build errors and (iv) CI is an important tool to assist the CR process in software projects. Our results can assist in improving the development process by optimizing the performance of CR and CI activities.
